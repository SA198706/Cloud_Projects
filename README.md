# Loan Default Prediction — Cloud-Based Data Science & ML Application

## Business Problem (Sub-Objective 1.1)

A bank approves loans without fully knowing repayment risk upfront.  
This application predicts the probability that a loan applicant will **default**
using historical Lending Club loan data, so underwriting can adjust approvals,
pricing, and loss reserves.

## Sub-Objective 1 — Data Pipeline (8 marks)

### 1.2 Data Ingestion

Dataset: **Lending Club Loan Data** (Kaggle — `wordsforthewise/lending-club`
or use the included synthetic generator for offline development).

```bash
# Option A — use the synthetic generator (no Kaggle account needed)
python scripts/generate_data.py        # → data/loan_data_raw.csv  (~10 000 rows)

# Option B — real Kaggle dataset
# Place loan.csv inside data/; preprocessing.py will detect it automatically.
```

### 1.3 Data Pre-processing (`scripts/preprocessing.py`)

- Summary statistics for all numeric columns (`df.describe()`)
- Data-type display
- Missing-value check (before and after imputation)
- Median imputation for numeric columns; mode imputation for categorical
- Z-score normalisation via `StandardScaler` (produces `*_scaled` columns)

```bash
python scripts/preprocessing.py       # standalone run; saves loan_data_clean.csv
```

### 1.4 Exploratory Data Analysis (`scripts/eda.py`)

- **Correlation coefficients** — Pearson matrix for all numeric features vs `default`
- **Binning** — `dti` → Low/Medium/High/Very High tiers; `annual_inc` → 4 income bands
- **Categorical encoding** — LabelEncoder for all categoricals (used in feature importance)
- **Feature importance** — RandomForestClassifier (200 trees, depth 6)
- **8 visualisations** saved to `plots/`:
  1. Class distribution (default vs non-default) — *univariate*
  2. Interest-rate distribution — *univariate*
  3. Annual-income distribution — *univariate*
  4. Default rate by loan grade — *bivariate*
  5. DTI by default status (boxplot) — *bivariate*
  6. Default rate by loan purpose — *bivariate*
  7. Correlation heatmap — *bivariate*
  8. Feature-importance bar chart — *bivariate*

```bash
python scripts/eda.py                  # standalone EDA + saves all 8 plots
```

Key findings:
- Default rate ≈ 26 % (realistic lending-risk calibration)
- `int_rate` is the top predictor (Pearson r ≈ 0.24; also #1 in Random Forest)
- Default rate rises monotonically A → G: A ≈ 17 %, G ≈ 60 %
- `dti` and `revol_util` are meaningful secondary drivers

### 1.5 DataOps — Automated Pipeline with Scheduling

#### Option A — APScheduler (local, no cloud account required)

```bash
python scripts/scheduler.py            # fires every 2 minutes; logs to scripts/logs/
python scripts/dashboard.py            # display run-history dashboard
python scripts/dashboard.py --watch    # auto-refresh every 30 s
python scripts/dashboard.py --tail 20  # show last 20 runs
```

Every run logs to `scripts/logs/pipeline_log.jsonl` (structured, machine-readable)
and `scripts/logs/pipeline.log` (human-readable).  The dashboard reads the JSONL
file and displays total runs, success rate, average default rate, per-run details,
and default rate by grade.

#### Option B — Prefect Cloud (cloud dashboard at app.prefect.cloud)

```bash
# 1. Authenticate once
prefect cloud login --key <your-api-key>

# 2. Full ML pipeline (data + EDA + train + evaluate), every 2 minutes
cd scripts
python ml_prefect_flow.py --serve      # registers + keeps running
```

Each `@task` is independently observable (logs, duration, retries,
success/failure) in the Prefect Cloud UI.

## Sub-Objective 2 — Machine Learning Pipeline (5 marks)

### 2.1 & 2.2 Model Preparation & Training (`scripts/model_training.py`)

Two algorithms selected:

| Algorithm | Rationale |
|---|---|
| **Logistic Regression** | Fast, interpretable linear baseline; handles class imbalance via `class_weight='balanced'` |
| **Random Forest** | Ensemble tree model; captures non-linear interactions; also `class_weight='balanced'` |

Dataset split: **70 % training / 30 % testing** (stratified by target).

```bash
python scripts/model_training.py       # trains both models; saves .pkl to scripts/models/
```

### 2.3 & 2.4 Model Evaluation & MLOps Monitoring (`scripts/model_monitor.py`)

**Five metrics logged per model:**

| Metric | Description |
|---|---|
| Accuracy | Overall correct predictions |
| Precision | True positives / (TP + FP) |
| Recall | True positives / (TP + FN) |
| F1 Score | Harmonic mean of precision and recall |
| ROC-AUC | Area under the ROC curve (primary best-model selector) |

Additional outputs: confusion matrix and full classification report.  
Best model selected by **ROC-AUC** (not raw accuracy, which is misleading on the
imbalanced ~80/20 dataset).  All metrics persisted to `scripts/metrics/model_metrics.json`.

```bash
python scripts/model_monitor.py        # evaluates both models; saves metrics + best_model.pkl
python scripts/ml_pipeline.py         # end-to-end: train + evaluate + save
```

#### Running the full ML Prefect flow (every 2 minutes):

```bash
cd scripts
python ml_prefect_flow.py --serve      # deploy + schedule every 2 min to Prefect Cloud
python ml_prefect_flow.py              # one-off local run
```
